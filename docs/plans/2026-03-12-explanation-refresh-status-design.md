# Explanation Refresh Status Design

## Goal

자동 갱신이 시작될 때 기존 상황 설명 본문이 이미 있으면, 본문을 `요청중` 메시지로 덮어쓰지 않고 이전 설명을 유지한다. 새 응답이 도착하면 그때 본문을 교체하고, 실패하면 이전 본문을 유지한 채 요청 상태만 별도로 표시한다.

## Current Problem

- 자동 갱신 요청을 제출할 때 `main.py`가 `explanation_result`를 즉시 `LOADING` 상태의 빈 결과로 덮어쓴다.
- `render_explanation_panel()`은 빈 본문 + `LOADING` 상태를 보고 `현재 장면 설명을 생성하는 중입니다.` 메시지를 렌더링한다.
- 그 결과, 유효한 이전 분석이 있어도 자동 갱신 중에 본문이 사라진다.

## Design

### 1. 본문 상태와 요청 상태를 분리

- 기존 `ExplanationResult` / `ExplanationStatus`는 "현재 패널에 보여줄 본문"의 상태로 유지한다.
- 별도 요청 상태 문자열을 추가한다: `idle`, `updating`, `failed`.
- 패널 상단에 `Refresh: ...` 줄을 추가해 요청 상태를 표시한다.

### 2. 자동 갱신 요청 제출 규칙

- 이전 성공 본문이 없으면 기존처럼 `LOADING` 본문으로 전환한다.
- 이전 성공 본문이 있으면:
  - `explanation_result`는 유지한다.
  - `explanation_status`도 유지한다.
  - 요청 상태만 `updating`으로 바꾼다.
  - 스크롤 위치도 유지한다.

### 3. 응답 처리 규칙

- 새 응답이 성공이고 본문이 비어 있지 않으면 본문을 교체하고 요청 상태를 `idle`로 바꾼다.
- 자동 갱신 실패 또는 빈 응답이 오고 이전 성공 본문이 있으면:
  - 이전 본문 유지
  - 요청 상태를 `failed`로 변경
- 이전 성공 본문이 없는 상태에서 실패하면 기존 오류 본문 표시를 유지한다.

## Testing

- 자동 갱신 중 이전 성공 본문이 유지되는지 확인
- 자동 갱신 실패 시 이전 본문이 유지되고 상태 줄만 `failed`로 바뀌는지 확인
- 패널 헤더에 `Refresh: ...` 줄이 렌더링되는지 확인
