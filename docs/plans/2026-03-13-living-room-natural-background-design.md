# Living Room Natural Background Design

**Problem**

`living_room_navigation_v1`의 `Environment Model` Open3D 창은 실내 구조와 가구는 보이지만, 전면 유리창 바깥이 단색 배경으로 비쳐서 외부가 자연 환경이라는 정보가 전달되지 않는다. 사용자는 창문 밖이 하늘, 잔디, 나무가 있는 자연 환경으로 인식되길 원한다.

**Decision**

시뮬레이션 충돌체나 Blender 카메라 렌더 경로는 건드리지 않고, `Environment Model` 전용 외부 배경 메시 레이어를 추가한다.

**Approach**

- `visualization.py`의 `Open3DEnvironmentViewer`가 장면 메시를 조합할 때, 창문 바깥에만 보이는 외부 배경 메시를 함께 생성한다.
- 외부 배경은 이미지 텍스처 대신 정점 색 기반 메시로 구성한다.
- 위쪽은 하늘 그라데이션 패널, 아래쪽은 잔디 지면 패널, 중간은 나무 수관과 줄기 실루엣 메시를 배치해 자연 환경을 암시한다.
- 배경 메시 위치는 전면 유리창(`front_glass`)보다 바깥쪽에 놓아 실내 메시와 시각적으로 분리한다.
- 기존 방/가구/카메라 프러스텀 렌더링과 뷰 설정은 유지한다.

**Data Model Changes**

새 시뮬레이션 상태 필드는 필요 없다. 배경 메시 생성에 필요한 기준점은 기존 `LivingRoomSceneSpec.room_size_xyz`와 전면 창 치수를 재사용한다.

**Tradeoffs**

- Open3D 전용 배경 레이어는 요구 범위에 가장 정확하게 맞지만, Blender 카메라 시점 RGB에는 같은 자연 배경이 반영되지 않는다.
- 이미지 텍스처 대신 정점 색 메시를 사용하므로 사실성은 제한되지만, 현재 뷰어 구조와 테스트 전략에는 가장 잘 맞는다.
- 외부 배경을 장면 스펙 전체에 넣지 않으므로 충돌체나 경로계획에는 영향이 없다.

**Success Criteria**

- `Environment Model` 창에서 전면 창문 바깥에 하늘이 보인다.
- 창문 아래/중간 영역에 잔디와 나무가 보여 실내 밖이 자연 환경임을 알 수 있다.
- 기존 실내 메시, 카메라 마커, 뷰어 동작은 유지된다.
- 시각화 테스트가 외부 배경 메시 포함 여부를 검증한다.
